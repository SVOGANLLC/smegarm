## Фаза 1 — Маркетинг и витрина магазина

Расширяю таблицу `products`:
- `is_featured`, `is_new`, `is_bestseller`, `is_special_offer` — флаги для блоков на главной
- `badge_text` — произвольная плашка («Хит», «-20%», «Новинка»)
- `price_old` — старая цена (вместо процента — честное «было/стало»)
- `promo_starts_at`, `promo_ends_at` — авто-активация/деактивация акций
- `sort_weight` — ручная сортировка в категории
- `seo_title`, `seo_description`, `slug`, `og_image` — SEO на товар
- `view_count` — счётчик для аналитики

Новые таблицы:
- `collections` — кураторские подборки (название, slug, обложка, видна/нет, порядок)
- `collection_products` — many-to-many товаров и коллекций
- `categories_meta` — SEO + баннер для каждой категории

На сайте:
- Главная: блоки «Хиты продаж», «Новинки», «Спецпредложения», «Коллекции»
- Бейджи на карточках товаров (хит, новинка, скидка, свой текст)
- Старая/новая цена со скидкой
- Страница `/sale` — все акции
- Страницы коллекций `/collection/$slug`
- Sitemap.xml + canonical + OG-теги на товары/категории

В админке:
- Чекбоксы флагов прямо в списке товаров и в форме редактирования
- Раздел «Коллекции»: создать, обложка, drag-and-drop товаров, сортировка
- Поля SEO в форме товара

**Корзина и оформление заказа:**

- Подтверждение заказа: страница «спасибо» + email клиенту + email админу + WhatsApp-ссылка

Таблицы: `orders`, `order_items`, `cart_items`.

**CRM-лайт для заявок и заказов:**
- Статусы (новая → в работе → подтверждена → выполнена / отменена)
- Назначить ответственного админа, заметки, история изменений
- Email админу при новой заявке/заказе (через Resend)

## Фаза 4 — Языки и контент

**3 языка: RU / HY / EN**
- Все UI-строки сайта и админки — через i18n (расширяю существующий `src/lib/i18n.tsx`)
- Переключатель языка в шапке (RU по умолчанию, локаль в URL: `/`, `/hy`, `/en`)
- Переводимые поля товаров: `name_ru/hy/en`, `description_ru/hy/en` (миграция: текущее значение → `_ru`, остальные пустые, fallback на RU)
- Переводимые названия категорий, коллекций, контент-блоков
- В админке — табы RU/HY/EN на каждой переводимой форме
- Auto-translate кнопка через Lovable AI (Gemini) — заполнить HY и EN из RU одним кликом, с возможностью редактирования
- SEO: `hreflang` теги, отдельные `sitemap-ru.xml` / `sitemap-hy.xml` / `sitemap-en.xml`

**Управляемые контент-блоки главной:**
- Hero-слайдер (несколько слайдов с заголовком, подзаголовком, кнопкой, картинкой)
- Баннеры между секциями
- Выбор: какие SKU крутятся в «Хитах» и «Новинках» (вручную или авто по флагам)
- Тексты разделов «О бренде», «Доставка», «Гарантия», «Контакты»
- Все редактируется в `/admin/content` с предпросмотром

## Технические детали (для меня, можно пропустить)

- Все серверные операции — `createServerFn` + `requireSupabaseAuth`, не Edge Functions
- Картинки на сервере ресайзит не sharp (нет в Workers), а Cloudflare Image Resizing через URL-параметры
- Email — Resend (нужен будет API-ключ позже, спрошу)
- WhatsApp — просто `wa.me/<номер>?text=...` ссылки, без API
- pg_cron для авто-снятия истёкших акций (через `/api/public/cron/expire-promos` с подписью)
- RLS: товары и контент — публичное чтение опубликованного, запись только админ; заказы — клиент видит свои, админ все
- Lovable Assets для всех фото коллекций/hero (не таскаем мегабайты в репо)
