# Smeg.am — Catalog & Navigation Roadmap

> **Цель:** функционал уровня shop.smeguk.com в **нашем визуальном стиле**, с умной фильтрацией по характеристикам — эталон для других представителей Smeg.

**Статус:** в работе · обновлено 2026-06-16

---

## Принципы (не отходим)

1. **Стиль smeg.am** — светлая премиальная эстетика, типографика, карточки. Берём только UX/функционал UK.
2. **Весь каталог + секции** — не взаимоисключающие режимы:
   - `/catalog` — **все товары** (по умолчанию, как сейчас).
   - Секции (Large / Small / Accessories) — **быстрый вход** через мега-меню и landing-ссылки; фильтруют подмножество, но пользователь всегда может вернуться в «Весь каталог» одним кликом.
   - Боковая панель: пункт **«Все»** всегда сверху списка категорий.
3. **Админка без кода** — навигация, группировка, фильтры настраиваются в `/admini`.
4. **Умные фильтры везде**, где список из нескольких товаров: каталог, коллекции, акции, поиск.

## Large / Small — авто-классификация

| Секция | Критерий |
|--------|----------|
| **Крупная техника** | `family` ∈ {Refrigerator, Oven, Hob, Cooker, Dishwashers, Sink, Microwave, Freezers, Washing Machine, …} **или** `category` из списка крупной техники |
| **Мелкая техника** | Kettles, Toaster, Blenders, Coffee machines, … |
| **Аксессуары** | Accessories, фильтры, решётки |
| **Fallback** | пустой `family` → `category_en` |

Правила хранятся в конфиге админки (редактируемые).

**URL-схема секций:** `/catalog?section=large` | `small` | `accessories` (не заменяет `/catalog`).

## Технические решения

### Нормализация specs

Specs в JSON с разными ключами (`Width`, `Product Width`, `Depth (mm)`). Решение:

```
spec_filter_fields  — реестр фильтров (slug, labels RU/EN/HY, type, aliases)
product_spec_values — индекс (sku, field_slug, value_text, value_num)
```

Заполнение: триггер на `products` + батч-скрипт при импорте.

### Контекстные фильтры

В «Духовках» не показываем «Загрузка белья»; в «Чайниках» — «Ниша встраивания». Правила: `spec_filter_fields.categories` / `.families`.

### Группировка цветов

`model_group` уже есть. Группировка в сетке — отдельный флаг в `site_content.catalog.config.groupByColor` (+ per-category overrides).
